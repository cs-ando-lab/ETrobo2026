# BLE 接続による走行体の内部状態の監視

本リポジトリでは、[**Pybricks BLE Monitor**](https://github.com/mhikichi-nkhs-gmail/pybricks-ble-monitor)を使用してPCから走行体の内部状態を監視できるようにしています。Pybricks BLE Monitor はインターネット接続不要のローカルWebページとして動作します。

## 3. BLE Monitor の使い方

### プログラムのビルドと転送

Gitで最新のリポジトリをプルしてから、`~/etrobo` で下記のコマンドを実行します。

```bash
make app=ETrobo2026_code/main up
```

本リポジトリには、USBのアタッチ・ビルド・転送を一括で行う `deploy.sh` を用意しています。`~/etrobo` に移動せずに、そのまま `~/etrobo/workspace/ETrobo2026_code` で下記のコマンドを実行することでもビルド＆転送が可能です。

```bash
./deploy.sh
```

### PCと実機をBLE接続する

#### Pybricks BLE Monitor を起動する

起動方法には、以下の2種類の方法があります。

- **`start.bat` から開く**
  1. VSCodeで `pybricks-ble-monitor/start.bat` を右クリックし、「エクスプローラーで表示する」をクリックする。
  2. `start.bat` をダブルクリックする。
- **`deploy.sh` による自動起動**

  `deploy.sh` は、プログラムの転送後にWindows側のChromeでPybricks BLE Monitorを開きます。すでに Pybricks BLE Monitor を開いているタブがある場合は、新しく開かずにそのタブを前面に表示します。

> [!NOTE]
> `index.html` を`file://`として直接ダブルクリックで開くと、ログの自動保存機能（`log`フォルダへの自動保存）が正しく動作しないことがあります。これは`file://`のURL形式によってはブラウザが一部の権限を許可しないためで、`start.bat`または`deploy.sh`を使うと、ローカルサーバー経由の`http://localhost`として開かれるため問題を回避できます。

#### PCと走行体を接続する

1. 起動した Pybricks BLE Monitor で「接続」ボタンをクリックする。
2. 表示されるウィンドウで「Pybricks Hub」を選択し、ペア設定をクリックする。

### 画面の見方

接続が完了すると、画面上部にセンサーごとの現在値が常に更新されながら表示されます。画面下部にはログ一覧が表示され、接続してから現在までの受信データを確認できます。各値の詳しい意味は [4. SPIKEセンサー取得値一覧](#4-spikeセンサー取得値一覧) を参考にしてください。

ログ表示は「**すべて**」タブと**センサーごと**のタブに分かれています。

- 「すべて」タブ: 受信しているすべてのセンサー値を表示します。
- 各センサーのタブ: そのセンサーの値のみに絞って表示し、列ヘッダー（`count`, `R`, `G`, `B` など）付きで見やすくなっています。右上の「グラフ」に切り替えると、時系列グラフとして表示できます。
- 一部の値は閾値を超えると赤色でハイライト表示されます（例: フォースセンサーが押されている、超音波センサーが測定不能で `-1` を返している、モータがストールしている、など）。

左パネルの「ログ操作」からは、ログを クリップボードへコピー／CSVで保存／テキストで保存できるほか、「ストリーミング保存」を有効にすると受信の都度ファイルへ追記していくこともできます。

## 4. SPIKEセンサー取得値一覧

SPIKEセンサーで取得可能な値の一覧を以下にまとめました。一部の値は出力可能ですが必要性が低いと判断し、BLE Monitorの出力には含めていません。「BLE Monitorでの監視」列は、現在BLE Monitorで監視している値かどうかを示しています。

### カラーセンサ（`ColorSensor.h` / `PORT_E`）

| メソッド                 | 戻り値 / 型                  | 単位                 | 説明                      | BLE Monitorでの監視 |
| ------------------------ | ---------------------------- | -------------------- | ------------------------- | ------------------- |
| `getRGB(rgb)`            | `RGB { r, g, b }` (uint16_t) | -                    | RGB値（各色8ビット）      | ✅                  |
| `getColor(hsv, surface)` | `HSV { h, s, v }`            | h: 0-360, s/v: 0-100 | 色をHSVで取得（近似あり） | ✅                  |
| `getHSV(hsv, surface)`   | `HSV { h, s, v }`            | h: 0-360, s/v: 0-100 | 色をHSVで取得（近似なし） | ✖                   |
| `getReflection()`        | `int32_t`                    | %                    | 表面の光反射率            | ✅                  |
| `getAmbient()`           | `int32_t`                    | %                    | 周囲光の強度              | ✖                   |

> `getColor` / `getHSV` の引数 `surface`：`true` で表面色、`false` で外部光源の色を検出します。
> カラーセンサーは対象物との距離が3cm程度離れると正確な値を取得できなくなるため注意してください。

### 超音波センサ（`UltrasonicSensor.h` / `PORT_F`）

| メソッド        | 戻り値 / 型 | 単位 | 説明                                | BLE Monitorでの監視 |
| --------------- | ----------- | ---- | ----------------------------------- | ------------------- |
| `getDistance()` | `int32_t`   | mm   | 対象物までの距離                    | ✅                  |
| `isPresence()`  | `bool`      | -    | 他の超音波信号の検出（`true`=検出） | ✖                   |

### フォースセンサ（`ForceSensor.h` / `PORT_D`）

| メソッド           | 戻り値 / 型 | 単位 | 説明                                                  | BLE Monitorでの監視 |
| ------------------ | ----------- | ---- | ----------------------------------------------------- | ------------------- |
| `getForce()`       | `float`     | N    | ボタンに加わる力                                      | ✅                  |
| `getDistance()`    | `float`     | mm   | ボタンの押し込み距離                                  | ✅                  |
| `isPressed(force)` | `bool`      | -    | 指定した力以上で押されているか（`true`=押されている） | ✖                   |
| `isTouched()`      | `bool`      | -    | ボタンに触れているか（`true`=触れている）             | ✅                  |

> `isPressed(force)` の引数 `force`：押されていると判定する最低限の力（N）

### IMU／慣性計測ユニット（`IMU.h` / HUB）

| メソッド                  | 戻り値 / 型                           | 単位  | 説明                           | BLE Monitorでの監視 |
| ------------------------- | ------------------------------------- | ----- | ------------------------------ | ------------------- |
| `getAcceleration(accel)`  | `Acceleration { x, y, z }` (float)    | mm/s² | X/Y/Z軸の加速度                | ✅                  |
| `getAngularVelocity(ang)` | `AngularVelocity { x, y, z }` (float) | °/s   | X/Y/Z軸の角速度                | ✅                  |
| `getTemperature()`        | `float`                               | ℃     | IMUモジュールの温度            | ✖                   |
| `getHeading()`            | `float`                               | °     | 方位角（ヨー角）               | ✅                  |
| `isReady()`               | `bool`                                | -     | IMUが使用可能か                | ✖                   |
| `isStationary()`          | `bool`                                | -     | IMUが静止状態か（`true`=静止） | ✖                   |

### モータ（`Motor.h` / `PORT_A`(右), `PORT_B`(左)）

| メソッド      | 戻り値 / 型 | 単位 | 説明                                            | BLE Monitorでの監視 |
| ------------- | ----------- | ---- | ----------------------------------------------- | ------------------- |
| `getCount()`  | `int32_t`   | °    | エンコーダの累積回転角度                        | ✅                  |
| `getSpeed()`  | `int32_t`   | °/s  | モータの現在回転速度                            | ✅                  |
| `getPower()`  | `int32_t`   | %    | 現在のパワー値（-100〜+100）                    | ✅                  |
| `isStalled()` | `bool`      | -    | ストール（脱調）しているか（`true`=ストール中） | ✅                  |

### バッテリ（`Battery` / HUB）

| メソッド       | 戻り値 / 型 | 単位 | 説明             | BLE Monitorでの監視 |
| -------------- | ----------- | ---- | ---------------- | ------------------- |
| `getCurrent()` | `uint16_t`  | mA   | バッテリーの電流 | ✖                   |
| `getVoltage()` | `uint16_t`  | mV   | バッテリーの電圧 | ✖                   |

### ボタン（`Button.h` / HUB）

| メソッド               | 戻り値 / 型 | 説明                            | BLE Monitorでの監視 |
| ---------------------- | ----------- | ------------------------------- | ------------------- |
| `isLeftPressed()`      | `bool`      | Leftボタンが押されているか      | ✅                  |
| `isCenterPressed()`    | `bool`      | Centerボタンが押されているか    | ✅                  |
| `isRightPressed()`     | `bool`      | Rightボタンが押されているか     | ✅                  |
| `isBluetoothPressed()` | `bool`      | Bluetoothボタンが押されているか | ✅                  |

各クラスに共通して `hasError()` メソッドがあり、デバイス取得の失敗を検出できます（`true` = エラーあり）。

## 5. 注意点

- **シャットダウン直前のログは送信されないことがあります。** `hub_system_shutdown()` を呼ぶ直前に `syslog()` を実行しても、送信前にBLE接続が切れてしまい受信できません。シャットダウン前に数百ms程度の待機（`dly_tsk()`）を入れることで正常に送信できます。
- **ポート指定ミスはアルファベット1文字として表示されます。** `PBIO_PORT_ID_A`〜`F` で間違ったポートを指定すると、そのポート名のアルファベットが表示されてエラーになります。どのポートにどのセンサーが接続されているか、`main/app.cpp` の初期化箇所を確認してください。
- **Pybricks BLE Monitor はMITライセンスであるため改変して本リポジトリに同梱しています。**
