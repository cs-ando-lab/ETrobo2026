# UMLモデル運用ルール

このドキュメントでは、astah\*で作成するUMLモデルの管理方法と、コードとの対応づけ方について説明します。

## 目次

## ファイル構成

```
models/
├── ETrobo2026.asta      # astah*プロジェクト本体（または .xml エクスポート）
└── diagrams/            # PRレビュー用にエクスポートした画像
    ├── class/            # クラス図
    ├── sequence/         # シーケンス図
    └── state/            # 状態遷移図
```

- `.asta`/`.xml` はバイナリに近くdiffが読めないため、変更時は必ず `diagrams/` 配下に **PNG または SVG を含める**。
- 画像ファイル名は図の名前と一致させる（例: `LineTracer状態遷移図` → `diagrams/state/line_tracer.png`）。

## 開発の流れ

1. astah\* でモデルを作成する。
2. `.asta`/`.xml`と画像ファイルをエクスポートし、コミット&プッシュを行う。
3. GitHub上でPRを作成する。
4. レビュアーの承認後、mainへマージを行う。

## PRでのレビュー方法

- レビュアーは `models/diagrams/` の画像と `main/` のコード差分を両方確認し、モデルとコードが矛盾していないかをチェックする。
